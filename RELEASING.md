# Releasing LAlatex

## Versioning

- `Project.toml` is the authoritative package version.
- Release tags must match the package version exactly:
  `1.0.0` -> `v1.0.0`.

## Pre-release verification

Run:

```powershell
julia --project=. -e 'using Pkg; Pkg.test()'
julia --project=docs docs/make.jl
```

Confirm:

- CI is green on the release commit.
- Docs are green on the release commit.
- The docs workflow uses the exact `actions/setup-python` interpreter path for
  `JULIA_PYTHONCALL_EXE`.
- Notebook smoke checks pass.
- The secondary CI OS lane is green.

## Backend note

`Backend.backend_usable(...)` is a runtime usability probe, not a static
installation check. For the SymPy backend it may initialize Python and attempt
to import `sympy`. Use `import_sympy()` when you want explicit initialization
or a direct import failure for diagnostics.

## Release steps

1. Update `Project.toml` to the target version.
2. Commit and push the release-preparation changes.
3. Wait for CI and Docs to complete successfully on `main`.
4. Create the matching tag (`vX.Y.Z`).
5. Create or update the GitHub Release object for that tag.
6. Replace autogenerated notes with a curated release summary.
